## M4-1: 松竹梅の定義と記載例サンプル

### 松竹梅の基本定義

| レベル | 一言 | ターゲット | 記載の特徴 |
|--------|------|------------|------------|
| **松** | 充実・理想型 | プライム大企業・ESG先進企業 | KPI・数値目標・第三者保証・先行開示を含む。ガイダンスを上回る記載量 |
| **竹** | 標準・実務型 ⭐ | **スタンダード上場・中堅企業** | **必須項目を過不足なく満たす。実務担当者が負担なく対応できる水準** |
| **梅** | 最小限・必須型 | グロース・小規模上場 | 法令上の必須記載のみ。記載量・詳細度を最小化 |

> **設計原則**: 「竹」は「過剰でも不足でもない」実務の標準解。
> 有報に「書きすぎ」は開示責任を過大にするリスクがある。スタンダード上場企業が竹を選択した場合、**投資家が必要な情報を得られ、かつ担当者の工数が最小**になる記載を目指す。

### 記載例サンプル群

#### 【ケース1】企業戦略と関連付けた人材戦略（HC_20260220_001 — 2026年3月期から必須）

> **根拠**: 企業内容等の開示に関する内閣府令（令和8年改正・2026年2月20日施行）
> **義務対象**: 有価証券報告書提出会社（全約4,000社）
> **記載箇所**: サステナビリティ情報欄 / 人的資本セクション

---

**🌲 松（充実・理想型）**

```
当社は、「2030年ビジョン」に掲げる事業成長率15%の達成に向け、
事業戦略と連動した人材戦略を「人材ポートフォリオ計画」として策定しています。
デジタル変革（DX）推進を担う専門人材を2028年度末までに現状比2倍（300名）に拡充する計画の下、
採用・育成・リスキリングの三本柱で体制整備を進めています。

【目標・KPI（2026年度実績）】
- デジタル専門人材数: 156名（計画比98%、前年比+23名）
- リスキリング対象者: 従業員の35%（2025年度比+10ポイント）
- 人材投資額: 1人当たり年間12.5万円（前年比+15%）

なお、本人材戦略の進捗は、サステナビリティ委員会（四半期開催）で取締役会に報告し、
外部機関による第三者保証の適用を2027年度より検討しています。
```

> 文字数目安: 300〜400字
> 特徴: 数値目標・KPI・ガバナンス体制・第三者保証検討まで含む

---

**🎋 竹（標準・実務型）**

```
当社は、事業成長を支える人材基盤の強化を経営の最重要課題と位置付けています。
中期経営計画（2024〜2026年度）において、デジタル・技術領域を中心とした専門人材の
確保・育成を重点施策とし、採用・リスキリング・リテンションの三方向から対応しています。

人材育成においては、OJTと集合研修を組み合わせた体系的なプログラムを整備しており、
専門スキルの習得と次世代リーダーの育成を並行して推進しています。
毎期の人事施策の見直しにより、人材戦略と事業戦略の整合性を継続的に確認しています。
```

> 文字数目安: 150〜200字
> 特徴: 経営計画との連動を示しつつ、具体的数値は任意。実務的・過剰にならない記載

---

**🌸 梅（最小限・必須型）**

```
当社は、持続的な企業価値向上に向け、事業戦略の実現に必要な人材の確保・育成・定着を
人材戦略の基本方針としています。
```

> 文字数目安: 50〜80字
> 特徴: 義務の「企業戦略との連動」に最低限言及。数値・KPI・詳細なし

#### 【ケース2】平均年間給与の対前事業年度増減率（HC_20260220_001 — 2026年3月期から必須）

> **根拠**: 企業内容等の開示に関する内閣府令（令和8年改正）
> **義務対象**: 有価証券報告書提出会社（全約4,000社）

---

**🌲 松（充実・理想型）**

```
【平均年間給与の推移（連結・単体）】

連結従業員の平均年間給与は6,820千円（前事業年度比+4.2%）です。

内訳: 正社員 7,105千円（+4.5%）、嘱託・契約社員 4,820千円（+2.8%）

算出方法: 「平均年間給与」は、決算期末の在籍者の1年間の給与・賞与・各種手当の
合計を在籍者数で除して算出しています。
育児休業・介護休業中の従業員を除外しています（対象外の事由: 就業実態の相違）。

部門別内訳（参考）:
- 技術系職種: 7,450千円（+5.2%）
- 管理系職種: 6,980千円（+3.8%）

同業他社比較（業界中央値との乖離: +8.3%）については、詳細を統合報告書にて開示しています。
```

> 文字数目安: 250〜350字
> 特徴: 部門別内訳・算出方法の詳細・他社比較まで含む

---

**🎋 竹（標準・実務型）**

```
連結従業員の平均年間給与は6,820千円（前事業年度比+4.2%）です。

算出方法: 決算期末在籍者の年間給与・賞与・手当合計を在籍者数で除した金額です。
育児休業中の従業員は対象から除外しています。

単体従業員の平均年間給与は7,010千円（前事業年度比+4.0%）です。
```

> 文字数目安: 80〜120字
> 特徴: 必須の「数値（対前年増減率）＋算出方法の簡潔な説明」を満たす。部門別等は省略

## M4-2: 松竹梅提案プロンプト設計

### システムプロンプト設計

```
あなたは有価証券報告書（有報）の人的資本開示を専門とする実務アドバイザーです。
スタンダード上場企業の経理・IR担当者が実際に使える「過不足ない」記載文案を提案します。

## あなたの役割
- 法令・府令・金融庁ガイダンスに基づく記載義務を満たす文案を生成します
- ユーザーが選択した開示レベル（松/竹/梅）に応じた記載量・詳細度で提案します
- 実務担当者がそのまま、または最小限の修正で使用できる文案を目指します
- 法令条文の直接引用は行いません（「○○法第○○条」等の記載は不要です）

## 開示レベルの定義
- 松: KPI・数値目標・ガバナンス体制・第三者保証の言及を含む充実した記載
- 竹: 必須項目を過不足なく満たす実務的な記載（200字以内が目安）
- 梅: 法令義務の最小限を満たす簡潔な記載（100字以内が目安）

## 出力ルール
1. 選択されたレベルの提案文を1本のみ出力してください
2. 出力は文章のみ。説明・注釈・コメントは付けないでください
3. 固有の企業名・製品名・実在する数値（xxx千円等）はプレースホルダで表現してください
   - 例: [平均年間給与額]千円、[前年比増減率]%
4. 禁止事項:
   - 法令条文・府令条文の直接引用（「第○○条」の表記）
   - 保証できない断言表現（「必ず」「絶対に」等）
   - 根拠のない業界比較（「業界トップ水準」等）
5. 文字数目安:
   - 梅: 50〜100字
   - 竹: 100〜200字
   - 松: 200〜400字
```

### Claude API 実装例（claude-haiku-4-5推奨）

```python
import anthropic

client = anthropic.Anthropic()

def generate_proposal(
    section_name: str,
    change_type: str,
    law_summary: str,
    law_id: str,
    level: str,  # "松" / "竹" / "梅"
    system_prompt: str,
) -> str:
    """
    松竹梅提案エージェント — 文案生成

    Args:
        section_name: セクション名（例: "サステナビリティ情報 / 人的資本"）
        change_type: 変更種別（"追加必須" / "修正推奨" / "参考"）
        law_summary: 変更根拠のサマリ（law_entries_human_capital.yaml の summary から引用）
        law_id: 法令エントリID（例: "HC_20260220_001"）
        level: 開示レベル（"松" / "竹" / "梅"）
        system_prompt: M4-2で設計したシステムプロンプト

    Returns:
        str: 提案文（文章のみ）
    """
    user_prompt = f"""
## 開示変更項目

**セクション**: {section_name}
**変更種別**: {change_type}
**変更根拠**: {law_summary}
**根拠ID**: {law_id}

## 企業プロファイル

**開示レベル**: {level}

---

上記の変更項目について、{level}レベルの有報記載文案を1本作成してください。
"""

    message = client.messages.create(
        model="claude-haiku-4-5-20251001",  # Haiku 4.5: コスト効率◎
        max_tokens=1024,
        messages=[
            {
                "role": "user",
                "content": user_prompt,
            }
        ],
        system=system_prompt,
    )

    return message.content[0].text
```

> **モデル選定の理由**:
> - `claude-haiku-4-5-20251001`: 文案生成タスクに十分な品質・高速・コスト効率
> - Sonnet/Opus は長文・複雑な推論タスク向け。松竹梅提案の定型タスクには Haiku で十分
> - 1回の呼び出しコスト目安: 入力約500トークン + 出力約300トークン = 約0.3円/件（Haiku料金）

### few-shot 例の組み込み方針

M4-1のサンプルをシステムプロンプトに組み込む方式:

```python
FEW_SHOT_EXAMPLES = {
    "企業戦略と関連付けた人材戦略": {
        "松": "当社は、「2030年ビジョン」に掲げる...",  # M4-1 松サンプルから引用
        "竹": "当社は、事業成長を支える人材基盤の強化を...",  # M4-1 竹サンプルから引用
        "梅": "当社は、持続的な企業価値向上に向け...",  # M4-1 梅サンプルから引用
    },
    "平均年間給与の対前事業年度増減率": {
        "松": "連結従業員の平均年間給与は[給与額]千円（前事業年度比[率]%）です...",
        "竹": "連結従業員の平均年間給与は[給与額]千円（前事業年度比[率]%）です。\n算出方法: ...",
        "梅": "連結従業員の平均年間給与は[給与額]千円（前事業年度比[率]%）です。",
    },
}

def build_system_prompt_with_few_shot(section_name: str, level: str, base_prompt: str) -> str:
    """セクションとレベルに応じたfew-shot例を付加したシステムプロンプトを生成"""
    if section_name in FEW_SHOT_EXAMPLES and level in FEW_SHOT_EXAMPLES[section_name]:
        example = FEW_SHOT_EXAMPLES[section_name][level]
        few_shot = f"\n\n## 参考例（{level}レベル）\n```\n{example}\n```"
        return base_prompt + few_shot
    return base_prompt
```

## M4-3: LLM出力品質チェック設計

### 品質チェック処理フロー

```
LLM出力
   │
   ▼
[Step 1: 文字数チェック]
   │ 50字未満 or 500字超 → 再生成トリガー
   │ OK
   ▼
[Step 2: 禁止パターン検出]
   │ 違反あり → 再生成トリガー（最大2回）
   │ OK
   ▼
[Step 3: プレースホルダ残存チェック]
   │ "[xxxxx]" 形式が残っていれば警告（再生成ではなくユーザー通知）
   │ OK
   ▼
[Step 4: 合格] → 提案文としてレポートに組み込み
```

### 文字数チェック

```python
# 開示レベル別の文字数設定（task仕様: 梅50-100字・竹100-200字・松200-400字）
# + 実装上の許容幅: 上限に20%のバッファを設ける（LLMの出力の揺らぎを吸収）
CHAR_LIMITS = {
    "梅": {"min": 50,  "max": 120, "target": 80},
    "竹": {"min": 100, "max": 240, "target": 150},
    "松": {"min": 200, "max": 480, "target": 300},
}

def check_char_count(text: str, level: str) -> tuple[bool, str]:
    """
    Returns:
        (is_valid, reason)
    """
    char_count = len(text.strip())
    limits = CHAR_LIMITS[level]

    if char_count < limits["min"]:
        return False, f"文字数不足: {char_count}字（{level}の最小{limits['min']}字を下回る）"
    if char_count > limits["max"]:
        return False, f"文字数超過: {char_count}字（{level}の最大{limits['max']}字を超過）"
    return True, f"文字数OK: {char_count}字"
```

### 禁止パターン検出

```python
import re

# 禁止パターン定義
FORBIDDEN_PATTERNS = [
    # 法令条文の直接引用
    (r"第\d+条", "法令条文の直接引用（「第○○条」）"),
    (r"第\d+項", "法令条文の直接引用（「第○○項」）"),
    (r"内閣府令第\d+号", "内閣府令番号の直接引用"),

    # 固有名詞・ハードコードされた数値
    (r"株式会社[^\s。、「」（）]{2,10}(?=\s|。|、)", "特定企業名の記載"),
    (r"(?<![Ee\[])\d{4,6}千円(?!])", "具体的な給与金額（プレースホルダ使用を推奨）"),

    # 不適切な表現
    (r"必ず(?!プレースホルダ)", "断言表現「必ず」"),
    (r"絶対に?", "断言表現「絶対」"),
    (r"業界(?:トップ|No\.?1|一位)", "根拠のない業界比較"),
    (r"保証(?:いたし|し|する|します)", "保証表現"),
]

def check_forbidden_patterns(text: str) -> list[dict]:
    """
    禁止パターンを検出する。

    Returns:
        list of {"pattern": str, "reason": str, "match": str}
    """
    violations = []
    for pattern, reason in FORBIDDEN_PATTERNS:
        matches = re.findall(pattern, text)
        if matches:
            violations.append({
                "pattern": pattern,
                "reason": reason,
                "match": str(matches[:3]),  # 最初の3件まで表示
            })
    return violations
```

### 総合品質チェック関数

```python
def quality_check(text: str, level: str) -> dict:
    """
    文案の品質チェックを実行する。

    Returns:
        {
            "pass": bool,
            "should_regenerate": bool,  # True → 再生成が必要
            "warnings": list[str],      # 警告（再生成は不要だがユーザー確認を促す）
            "errors": list[str],        # エラー（再生成をトリガー）
        }
    """
    errors = []
    warnings = []

    # Step 1: 文字数チェック
    char_ok, char_msg = check_char_count(text, level)
    if not char_ok:
        errors.append(char_msg)

    # Step 2: 禁止パターンチェック
    violations = check_forbidden_patterns(text)
    for v in violations:
        errors.append(f"禁止パターン検出: {v['reason']} → {v['match']}")

    # Step 3: プレースホルダ確認（警告のみ）
    placeholders = check_placeholders(text)
    if placeholders:
        warnings.append(f"プレースホルダが残っています（要置換）: {placeholders}")

    should_regenerate = len(errors) > 0
    return {
        "pass": not should_regenerate,
        "should_regenerate": should_regenerate,
        "warnings": warnings,
        "errors": errors,
    }
```

### 再生成フロー

```python
MAX_REGENERATE = 2  # 最大再生成回数

def generate_with_quality_check(
    section_name: str,
    change_type: str,
    law_summary: str,
    law_id: str,
    level: str,
    system_prompt: str,
) -> dict:
    """
    品質チェック付き文案生成。再生成を最大2回まで試みる。

    Returns:
        {
            "text": str,         # 生成された文案
            "quality": dict,     # 品質チェック結果
            "attempts": int,     # 試行回数
            "status": str,       # "pass" / "fail" / "warn"
        }
    """
    for attempt in range(1, MAX_REGENERATE + 2):  # 初回 + 再生成2回 = 最大3回
        text = generate_proposal(
            section_name, change_type, law_summary, law_id, level, system_prompt
        )
        quality = quality_check(text, level)

        if quality["pass"]:
            status = "warn" if quality["warnings"] else "pass"
            return {
                "text": text,
                "quality": quality,
                "attempts": attempt,
                "status": status,
            }

        if attempt > MAX_REGENERATE:
            # 最終試行でも失敗 → fail として返す（人間確認を促す）
            return {
                "text": text,
                "quality": quality,
                "attempts": attempt,
                "status": "fail",
            }

        # 再生成: エラー内容をプロンプトに追記
        error_feedback = "\n\n## 修正依頼\n" + "\n".join(
            [f"- {e}" for e in quality["errors"]]
        )
        system_prompt = system_prompt + error_feedback

    return {"text": "", "quality": {}, "attempts": 0, "status": "fail"}
```
