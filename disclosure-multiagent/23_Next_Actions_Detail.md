## 2. PDF解析の PoC 実施

### 目的
様式第19号の有報 PDF からテキスト・セクションを抽出する手段を選定する。

### 比較対象

| 手段 | 特徴 | 検証ポイント |
|------|------|--------------|
| **PyMuPDF (fitz)** | 軽量、ローカル、テーブル抽出は別途 | テキスト抽出精度、見出しパターンでの分割 |
| **pdfplumber** | テーブル抽出に強い | 同上 |
| **Document AI** | 高精度、クラウド | API 利用可否、コスト、機密性 |
| **pymupdf4llm** | PyMuPDF の LLM 向け前処理 | 構造化出力のしやすさ |

### 検証手順

1. **サンプル確保**: EDINET から公開有報 PDF を 3〜5 社取得（スタンダード上場、人的資本記載あり）
2. **抽出テスト**: 各手段でテキスト抽出。人的資本セクションが取得できるか確認
3. **評価**: 抽出精度、処理時間、セクション分割のしやすさを比較
4. **選定**: Phase 1 の主手段を決定。フォールバック手段も記録

### 成果物

- **レポート**: `10_Research/PDF_PoC_Result.md`
- **検証スクリプト**: `scripts/pdf_poc_extract.py`
- **手順**: `10_Research/PDF_PoC_README.md`, `10_Research/EDINET_サンプル取得手順.md`
- **内容**: 比較表、選定理由、サンプルコード（またはリポジトリへのリンク）

### ブロッカー

- サンプル有報 PDF の確保（EDINET からダウンロード）

### 所要目安
1〜2 週

## 3. 法令情報の YAML 管理フォーマット設計

### 目的
7.3 冗長性担保に準拠した YAML スキーマを定義する。1 改正＝1 エントリ、Git 管理、起動時チェックを想定。

### スキーマ（案）

```yaml
# laws/disclosure_fiscal_2024.yaml 等
version: "1.0"
effective_period:
  from: "2024-04-01"
  to: "2025-03-31"

amendments:
  - id: "jc-2024-001"
    category: "人的資本"
    change_type: "追加必須"  # 追加必須 | 修正推奨 | 参考
    effective_from: "2024-04-01"
    title: "人的資本開示ガイダンス 改訂"
    source: "https://www.fsa.go.jp/..."
    summary: "平均給与増減率の算出方法を明確化"
    target_sections: ["経営方針", "人的資本"]
    required_items: ["平均給与増減率", "算出方法"]
```

### 必須フィールド

| フィールド | 説明 |
|------------|------|
| `id` | 一意ID（改正ごと） |
| `category` | 人的資本 / SSBJ / 総会前開示 等 |
| `change_type` | 追加必須 / 修正推奨 / 参考 |
| `effective_from` | 施行日 |
| `source` | 出典 URL |
| `target_sections` | 該当有報セクション |
| `required_items` | 必須記載項目（任意） |

### 重要カテゴリの必須チェック

起動時、対象期間内に以下のカテゴリでエントリが 1 件以上あるか検証する。

- `人的資本`（Phase 1 必須）
- `SSBJ`（将来対象企業向け。Phase 1 は任意）

### 成果物

- **スキーマ定義**: `docs/law_yaml_schema.md` または README
- **サンプル YAML**: 人的資本 5〜10 件のエントリ
- **チェックロジック**: 重要カテゴリ検証の疑似コード or 実装

### 所要目安
3〜5 日
