# OSS 公開可否 厳格レビュー

**日付**: 2025年3月  
**結論**: **push 可能だが、以下の「必須対応」を完了し、「推奨対応」は公開前または直後に済ませることを推奨する。**

---

## 1. 必須項目（これがないと OSS として不適切）

| 項目 | 状態 | 備考 |
|------|------|------|
| **LICENSE** | ✅ 対応済 | `LICENSE`（MIT）をリポジトリルートに追加。著作権表記は「Copyright (c) 2025 nencho contributors」。実名で運用する場合は 1 行目を差し替えること。 |
| **README** | ✅ 問題なし | 想定ユーザー・非対応・免責・インストール・API 概要が揃っている。 |
| **個人情報・秘密の混入** | ✅ 問題なし | コード・ドキュメントにハードコードされたパスワード・API キー・実名データはなし。テスト用の `_PASSWORD = "test_pass_*"` はテスト専用で許容範囲。 |
| **.gitignore** | ✅ 問題なし | `nencho.db`、`*.nchbk`、`output/`、`security_settings.json`、`.env` 等を除外。個人情報が誤ってコミットされない構成。 |
| **ログに秘密を出さない** | ✅ 問題なし | 認証失敗時は `db_path_str` のみログ。パスワード・個人データはログ出力していない。 |

## 3. コード・仕様上の指摘（公開可能だが認識しておくべき点）

| 項目 | 内容 | 推奨 |
|------|------|------|
| **税制の要確認** | `src/core/calculation/deductions.py` に「655万超 ≤ 2400万」の **閾値 2350万 or 2400万は要確認** とコメントあり。 | 国税庁・施行令で確認し、コードとコメントを確定。未確定のままでも push は可だが、README または capabilities に「基礎控除の高所得区分は国税庁等で要確認」と 1 行書くと安心。 |
| **対応年度が R7 のみ** | `TaxYear` は令和7年（2025年）のみ。令和8年分は未実装。 | README に「令和7年分を正式対応」と明記済み。R8 は今後の拡張として問題なし。 |
| **API 認証** | `POST /api/calculate` は認証なし。従業員系は `db_path` + `password`。 | 設計として許容。SECURITY.md に「本番で API を公開する場合は認証・HTTPS を推奨」と記載済み。 |

## 8. 総合判定

| 観点 | 判定 |
|------|------|
| **法的・ライセンス** | ✅ LICENSE あり。push 可。 |
| **秘密・個人情報** | ✅ 漏れなし。.gitignore とログ方針で問題なし。 |
| **ドキュメント** | ✅ README・SECURITY・スキル・免責が揃っている。プレースホルダーも明記済み。 |
| **品質・保守性** | △ テスト通過・税制 1 点要確認あり。CI はあるとよい。 |
| **初回リリースとして** | ✅ **push してよい水準。** 上記「必須」は満たし、「推奨」のうち SECURITY と README は対応済み。CONTRIBUTING・CI・税制の 1 点は公開後でも対応可能。 |

**結論（再掲）**: **必須対応は完了している。LICENSE と SECURITY.md を追加し、README のプレースホルダーを明記したうえで、OSS として git push してよい。** 公開後、CONTRIBUTING・CI・基礎控除の根拠確認を順次行うとよい。
